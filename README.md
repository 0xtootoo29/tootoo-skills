# Skill Router

智能 Skill 路由器 - 通过简洁的交互式问答，帮助用户在众多相似 skills 中找到最合适的那个。

## 为什么需要 Skill Router？

当你安装了大量 Claude Code Skills 后，会遇到这些问题：
- 🤔 记不清每个 skill 的具体名称
- 🔀 同类型的 skills 太多，不知道该用哪个
- 📝 想"写文章"，但有 4 个写作相关的 skills
- 🎨 想"生图"，但有封面、漫画、配图等多种选择
- 🚀 不想记住斜杠命令，只想说"我要做什么"

**Skill Router 就是为了解决这些问题而生的。**

## 核心特性

### 1. 🎯 智能意图识别
你只需说"写文章"、"生图"、"发布内容"，系统自动识别你的意图。

### 2. 📋 交互式选项卡
不需要在聊天框里打字回复，直接点选选项即可。

### 3. 🔄 动态识别新 Skills
每次运行时自动扫描 skills 目录，新安装的 skills 自动纳入路由。

### 4. 📐 平台规则展示
调用时自动展示平台特定规则（如 X、小红书、公众号的图片比例）。

### 5. 🧠 学习用户偏好
记住你的口语习惯和常用选择，下次优先推荐。

### 6. 🎨 简洁友好
- 隐藏技术细节，只在最后展示 skill 名称
- 渐进式提问，每次只问一个关键问题
- 用通俗语言描述功能

## 使用场景

### 场景 1：图像生成
```
你："生图"
系统：[选项卡] 您想要生成什么类型的图片？
      (a) 为平台设计封面/海报
      (b) 创作知识漫画
      (c) 为文档生成配图

你：选择 (a)
系统：[选项卡] 您想要为哪个平台设计？
      (a) X (Twitter) - 16:9 或 1:1
      (b) 小红书 - 3:4 竖图
      (c) 公众号 - 16:9
      (d) YouTube - 16:9

你：选择 (a)
系统：✅ 为您推荐：海报/封面设计工具 (`poster-design`)

      X 平台图片规则：
      - 推荐比例：16:9 (1200x675)
      - 格式：JPG, PNG
      - 大小：<5MB

      🚀 我现在就帮您启动 `poster-design`？
```

### 场景 2：写作创作
```
你："写文章"
系统：[选项卡] 您想要？
      (a) 从零开始创作内容
      (b) 编辑已有文本
      (c) 提取内容洞察

你：选择 (a)
系统：[选项卡] 您更喜欢哪种创作方式？
      (a) 采访式引导（通过问答引导，适合快速创作）
      (b) 深度思考大纲（哲学思考，构建高质量大纲）

你：选择 (a)
系统：✅ 为您推荐：采访式内容创作工具 (`content-creator`)

      这个工具会通过半结构化采访，将您的灵感、素材转化为可分发的内容。

      🚀 我现在就帮您启动 `content-creator`？
```

### 场景 3：不确定要做什么
```
你："帮我选个 skill"
系统：[选项卡] 您想要完成什么类型的任务？
      (a) 图像生成（封面、海报、漫画、配图）
      (b) 写作创作（文章、内容创作）
      (c) 内容分发（发布到社交平台）
      (d) 开发工具（API、前端、数据库）
      (e) 知识管理（调研、下载、转换）
      (f) AI 工具管理（Skill、MCP、CLAUDE.md）

[然后根据你的选择继续细化...]
```

## 平台规则库

Skill Router 内置了主流平台的规格要求：

### 图片规格
- **X (Twitter)**: 1200x675 (16:9) 或 1200x1200 (1:1), <5MB
- **小红书**: 1080x1440 (3:4), <10MB
- **公众号**: 900x500 (16:9), <2MB
- **YouTube**: 1280x720 (16:9), <2MB
- **Instagram**: 1080x1080 (1:1), <30MB

### 文字规则
- **X (Twitter)**: 280 字符（中文约 140 字）
- **小红书**: 标题 20 字，正文 1000 字
- **公众号**: 标题 64 字，正文无限制
- **YouTube**: 标题 100 字符，描述 5000 字符

## 智能特性

### 动态扫描
每次运行时自动扫描 `~/.claude/skills/` 目录，新安装的 skills 自动出现在选项中。

### 学习偏好
记录你的选择历史和自定义触发词：
- 你说"搞个图" → 系统学习这是"图像生成"
- 你常选择 X 平台 → 下次优先推荐 X 平台选项

### 智能分类
根据 skill 的 description 和 trigger_words 自动分类：
- 包含 "image", "poster", "comic" → 图像生成类
- 包含 "write", "content", "article" → 写作创作类
- 包含 "publish", "post", "platform" → 内容分发类

## 工作原理

1. **识别意图**：分析用户输入，判断属于哪个大类
2. **扫描 Skills**：实时扫描 skills 目录，筛选相关 skills
3. **渐进式提问**：通过 AskUserQuestion 提供选项卡
4. **展示规则**：如果涉及平台，展示平台规则
5. **推荐 Skill**：显示通俗描述 + 技术名称
6. **记录偏好**：保存用户选择，优化未来推荐

## 设计理念

### 用户友好优先
- ❌ 不展示技术术语（"识别意图"、"扫描结果"）
- ❌ 不列出 skill 名称列表（除非在最后推荐）
- ✅ 直接问用户想要做什么
- ✅ 使用选项卡而不是文本回复

### 渐进式交互
- 第一层：问大方向（从零创作 vs 编辑文本）
- 第二层：问细节（平台、风格、用途）
- 第三层：推荐 skill 并询问是否调用

### 透明可控
- 最后展示 skill 技术名称（如 `content-creator`）
- 用户知道具体调用了哪个 skill
- 方便后续针对性改进

## 适用人群

- 🆕 **新手**：不熟悉 skills，需要引导
- 🚀 **效率党**：不想记住所有 skill 名称
- 🎯 **选择困难症**：同类 skills 太多，不知道选哪个
- 🧠 **自然交互爱好者**：喜欢说"我要做什么"而不是记命令

## 与其他 Skills 的关系

Skill Router 不是替代其他 skills，而是帮助你找到它们：
- 如果你熟悉某个 skill，直接用 `/skill-name` 调用
- 如果你不确定用哪个，用 Skill Router 来引导
